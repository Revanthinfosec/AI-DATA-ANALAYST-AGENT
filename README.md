# 📊 AI Data Analyst Agent

An agentic data analysis tool powered by **Claude Sonnet 4.5**. Upload any dataset and get automated profiling, AI-generated insights, smart visualizations, and a data-aware business advisor chat — all in one place.

## ✨ Features

| Feature | Description |
|---|---|
| 📁 **Multi-format upload** | CSV, Excel (.xlsx), JSON, Parquet |
| 📋 **Auto profiling** | Shape, nulls, dtypes, duplicates, correlations |
| 💡 **AI insights** | 5–8 specific insights generated by Claude |
| 📈 **Smart charts** | Histograms, boxplots, heatmaps, line charts, scatter plots |
| 💬 **Business advisor chat** | Ask any question grounded in your actual data |
| 🔍 **NL→SQL** | Plain-English questions converted to DuckDB SQL |
| 🗂️ **Multi-file support** | Upload multiple files — each gets its own tab |
| 🔒 **Password gate** | Optional password to protect your API credits |

## 🚀 Quick Start

### 1. Clone the repo

```bash
git clone https://github.com/Revanthinfosec/AI-DATA-ANALAYST-AGENT.git
cd AI-DATA-ANALAYST-AGENT
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Set up environment variables

```bash
cp .env.example .env
```

Edit `.env` with your API credentials:

```env
ANTHROPIC_API_KEY=your-api-key-here
ANTHROPIC_BASE_URL=https://api.blackbox.ai
MODEL_NAME=claude-sonnet-4-5-20250514
SANDBOX_TIMEOUT=30
MAX_CSV_ROWS=100000
CHART_OUTPUT_DIR=/tmp/charts/
```

### 4. Run the app

```bash
# Streamlit UI (recommended)
streamlit run app.py

# CLI mode
python app.py --file sample_data/example.csv

# CLI with NL→SQL interactive mode
python app.py --file sample_data/example.csv --sql

# Analyse multiple files at once
python app.py --file sales.csv customers.xlsx orders.parquet
```

## 💬 Business Advisor Chat

The chat feature is what makes this different from a standard profiler. It gives Claude full context of your data — column names, distributions, correlations, and pre-generated insights — so every answer is **specific to your numbers**, not generic advice.

**Example questions you can ask:**
- *"How can I grow revenue in the North region?"*
- *"Which product has the best performance?"*
- *"What are the biggest risks I should address?"*
- *"What would happen to revenue if return rates doubled?"*
- *"Which customer segment should I focus on?"*

## ⚙️ Environment Variables

| Variable | Required | Default | Description |
|---|---|---|---|
| `ANTHROPIC_API_KEY` | ✅ | — | Your Blackboxai or Anthropic API key |
| `ANTHROPIC_BASE_URL` | ✅ | `https://api.blackbox.ai` | API base URL |
| `MODEL_NAME` | ✅ | `claude-sonnet-4-5-20250514` | Model identifier |
| `SANDBOX_TIMEOUT` | ❌ | `30` | Code execution timeout (seconds) |
| `MAX_CSV_ROWS` | ❌ | `100000` | Max rows to load |
| `CHART_OUTPUT_DIR` | ❌ | `/tmp/charts/` | Chart save location |
| `APP_PASSWORD` | ❌ | — | Password gate for deployed app |

## 🌐 Deploy to Streamlit Cloud

1. Fork this repo
2. Go to [share.streamlit.io](https://share.streamlit.io) → **New app**
3. Select your repo, branch `main`, file `app.py`
4. Click **Advanced settings → Secrets** and add:

```toml
ANTHROPIC_API_KEY = "your-key"
ANTHROPIC_BASE_URL = "https://api.blackbox.ai"
MODEL_NAME = "claude-sonnet-4-5-20250514"
APP_PASSWORD = "your-password"
```

5. Click **Deploy**

## 🔒 Security

- LLM-generated code runs in a **subprocess sandbox** — never `exec()` in the main process
- Sandbox writes are restricted to `/tmp/sandbox/`
- Execution killed after `SANDBOX_TIMEOUT` seconds
- API key protected via environment variables (never committed to git)
- Optional `APP_PASSWORD` gate for public deployments
